Vanguard — Autonomous eVTOL Testbed

Vanguard is Valkyrie's eVTOL research program. I drive hands-on hardware and flight-software development on a quad-rotor prototype testbed — architected so the components, sensors and autonomy code transfer up to a full-scale aircraft. I select the avionics, wire the hardware, author the firmware configuration, and write the autonomy software that runs on top of it.

The current flight stack pairs a Pixhawk 6X Pro flight controller with an NVIDIA Jetson Orin companion computer over MAVLink, with an external compass, downward lidar, downward optical flow and a forward camera. I set the Jetson up as a full on-aircraft development environment — running the toolchain and AI-assisted development directly on the companion computer.

Three-layer obstacle avoidance

The flagship feature is a defence-in-depth avoidance architecture — a velocity-governed "safety pillow" so the vehicle can't be flown into an obstacle at any speed. Each layer stands on its own, and the novel logic lives on the companion computer so the autopilot stays the certified-style safety core (no firmware fork).

Layer 1

Flight-controller safety floor

Obstacle sensors wired directly into the Pixhawk, driving ArduPilot's proximity / AVOID subsystem. Deterministic, and functional even if the companion computer drops offline.

ArduPilot AVOIDProximityDeterministic
Layer 2

Jetson perception

A depth-camera computer-vision pipeline builds a local 3D obstacle map on the Jetson and streams it to ArduPilot as OBSTACLE_DISTANCE_3D MAVLink messages at ≥ 10 Hz.

Depth CV3D mapMAVLink ≥10Hz
Layer 3

Velocity governor

A physics-based braking envelope, v_max = √(2·a·(range − margin)), shapes velocity setpoints so the aircraft always has room to stop — logging every intervention.

Braking envelopeSetpoint shapingTelemetry

I performed a sensor-suite gap analysis against the braking-envelope math and specified the additions to close it — 360° scanning lidar for Layer 1, a forward depth camera for Layer 2, and an upward time-of-flight rangefinder — then benchmarked flight-controller and firmware choices (ArduPilot vs PX4, triple-redundant IMU autopilots) on transferability to the full-scale build.
