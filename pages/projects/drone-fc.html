Custom STM32 Flight Controller

View on GitHub

A quadcopter built from the flight controller up around a bare STM32H743IIT6 board (FK743M2-IIT6) — the same MCU family that powers commercial Pixhawk autopilots. Rather than buy an off-the-shelf board, I brought up the hardware myself and ran ArduPilot on it, which meant understanding the autopilot from the pin map upward.

The core of the project is a hand-authored ArduPilot hwdef.dat — the board-definition file that tells the firmware exactly what is wired where. Getting sensors, radio and motors talking to the autopilot is the same flight-software stack I work in professionally, so this build doubles as a personal proving ground for the tools behind autonomous flight.

Board bring-up

Compute

  • STM32H743IIT6 MCU
  • FK743M2-IIT6 board
  • Bare-metal, no dev board

Sensors & radio

  • ICM-42688-P IMU (SPI2)
  • BMP388 barometer (I2C1)
  • FS-iA6B iBUS RX (UART2)

Actuation

  • TIM1 PWM → 4 ESCs
  • X-frame motor order
  • Full power / FPV chain

Why ArduPilot

I chose ArduPilot over PX4 after a driver- and ecosystem-level trade-off analysis — weighing peripheral support, the maturity of its board-definition workflow, and how cleanly custom logic could sit alongside the certified-style safety core. It's the same firmware family I use on the eVTOL testbed I work on professionally, so lessons move directly between the two.
